- profiles: id, role (brand|creator|admin|agency), display_name
- creators: platform (youtube|instagram|tiktok), followers, subscribers, engagement_rate, geo_country
- brands: name, slug, logo_url, is_verified, category_id
- campaigns: brand_id, title, brief, platforms (text[]), deal_type (paid_post|affiliate|gifting|ambassador), payout_model (flat|commission|product_cash), payout_amount, budget_total, status (draft|open|paused|closed|filled), deadline, min_followers, min_engagement_rate
- campaign_applications: campaign_id, creator_id, status (applied|shortlisted|accepted|rejected|withdrawn), bid_amount, message
- deals: campaign_id, brand_id, creator_id, price, status (active|submitted|approved|completed|cancelled|disputed), deadline

### File locations
- lib/supabase/client.ts — createBrowserClient
- lib/supabase/server.ts — createServerClient (async)
- All dashboard pages use server component for auth check, then pass initials to client component

### Quick reference for each upcoming screen

**Screen 3a — Brand Campaign Manager:** list brand's own campaigns with status badges, applicant count, budget. Pull from `campaigns` table filtered by `brand_id`.

**Screen 3b — Create Campaign:** form with all campaign fields. Inserts into `campaigns` with `brand_id` from brand profile.

**Screen 3c — Creator Applications:** list creator's own applications with campaign title, brand name, bid amount, status. Join `campaign_applications` + `campaigns` + `brands`.

**Screen 4 — Deals:** active deals for both sides. Join `deals` + `campaigns` + `brands/creators`.

**Screen 5 — Messages:** conversations list + message thread. Uses `conversations` + `messages` tables (already in DB).
